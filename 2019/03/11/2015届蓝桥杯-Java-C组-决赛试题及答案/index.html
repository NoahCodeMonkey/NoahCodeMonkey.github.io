2015年省赛的题目已经一遍过了下来，遗憾的就是最后一条还遗留在那里，等我感觉时间差不多了，我会回头将他解决的，加油！！！

机器人数目

题目

少年宫新近邮购了小机器人配件，共有3类，其中，
A类含有：8个轮子，1个传感器
B类含有: 6个轮子，3个传感器
C类含有：4个轮子，4个传感器

他们一共订购了100套机器人，收到了轮子600个，传感器280个。
根据这些信息请你计算：B类型机器人订购了多少个？

请直接提交该整数，不要填写任何多余内容。

思路

这道题很简单，相当于解一个三元一次方程，不过又让我想到一开始学C语言时的百元买百鸡的问题，很类似。

答案

60

生成回文数

题目

所谓回文数就是左右对称的数字，比如：
585,5885,123321…
当然，单个的数字也可以算作是对称的。

小明发现了一种生成回文数的方法：
比如，取数字19，把它与自己的翻转数相加：
19 + 91 = 110，如果不是回文数，就再进行这个过程：
110 + 011 = 121 这次是回文数了。

200以内的数字中，绝大多数都可以在30步以内变成回文数，只有一个数字很特殊，就算迭代了1000次，它还是顽固地拒绝回文！

请你提交该顽固数字，不要填写任何多余的内容。

思路

这道题看上去非常简单，我回文数都做过好几道了，一开始感觉一点问题都没有，当做之后才发现并没有想象中的那么简单，到处报错或者求不到结果，其中设置的值我从 int 转变成 long，在最后将值的类型转变成 BigInteger 才成功了，简简单单的一道题目耗了我半天时间，一开始还想法用递归，最后绕着绕着总感觉哪儿不对，最后从网上找到一个大佬的代码，自己在多写了一个 test方法就行了，真是服了自己了，很简单的题目就多写一个方法就行的事，非要搞那么复杂。。。

在此处在标记一个知识点：

BigInteger 的比较是使用 compareTo() 方法，若相等返回0，小于返回-1，大于则返回1

答案

196

空心菱形

题目

小明刚刚开发了一个小程序，可以打印出任意规模的空心菱形，规模为6时，如下图：

（如果有对齐问题，参看【图1.png】）

他一高兴，踢掉了电源，最后一次修改没有保存…..
毛病出在划线的部分。
请你帮助小明分析程序，填写划线部分缺失的代码。

public class Main
{
static String pr(int m, int n)
{
String s = “”;
for(int i=0; i<n; i++) s += “ “;
for(int i=0; i<m; i++) s = ““ + s + ““;
return s;
}

static void f(int n)
{
    String s = pr(1,n*2-1) + "\n";
    String s2 = s;

    for(int i=1; i<n; i++){
        s = ____________________________________;  //填空位置
        s2 = s + s2 + s; 
    }

    System.out.print(s2);
}

public static void main(String[] args)
{
    f(6);
}

}

注意：只填写缺少的内容，不要填写题面已有代码或说明性文字。

思路

唉，这道题还是看了一下别人的代码才知道的，之前一直总想着 s 要加什么才能完成这个空心菱形，可怎么试都不行，最后看了一下别人的一看到别人用到了 pr 方法就瞬间想通了，原来这个图形是 s2 为最中间一行的样式 s 则为上下两边的样式，立马去找 * 号和空格与行数 i 之间的关系就可以完成本题了，以后思想还是不要太固话，填空题中如果提供了一些方法一定要去想想是不是可以借助它的方法去完成本道题目。

答案

pr(i + 1, 2 n - 1 - 2 i) + “\n”

奇怪的数列

题目

从X星截获一份电码，是一些数字，如下：

YY博士经彻夜研究，发现了规律：
第一行的数字随便是什么，以后每一行都是对上一行“读出来”
比如第2行，是对第1行的描述，意思是：1个1，1个3，所以是：1113
第3行，意思是：3个1,1个3，所以是：3113

请你编写一个程序，可以从初始数字开始，连续进行这样的变换。

数据格式：

第一行输入一个数字组成的串，不超过100位
第二行，一个数字n，表示需要你连续变换多少次，n不超过20

输出一个串，表示最后一次变换完的结果。

例如：
用户输出：
5
7

则程序应该输出：
13211321322115

资源约定：
峰值内存消耗（含虚拟机） < 512M
CPU消耗 < 1000ms

请严格按要求输出，不要画蛇添足地打印类似：“请您输入…” 的多余内容。

所有代码放在同一个源文件中，调试通过后，拷贝提交该源码。
注意：不要使用package语句。不要使用jdk1.7及以上版本的特性。
注意：主类的名字必须是：Main，否则按无效代码处理。

思路

一开始看到题目想法就有了，但是感觉要用到递归，瞬间感觉有点吃力，尝试做了一下，各种强转什么 string 转 char 数组，在将值存入 set 中找到排除道重复值，然后将值取出与 char 数组进行比较进行计算个数，再递归调用这方法把新的值在处理，但感觉好麻烦做到中间强转就受不了了，然后就从网上搜索，看了一下别人的代码那么短，自闭了。。。。

密文搜索

题目

福尔摩斯从X星收到一份资料，全部是小写字母组成。
他的助手提供了另一份资料：许多长度为8的密码列表。
福尔摩斯发现，这些密码是被打乱后隐藏在先前那份资料中的。

请你编写一个程序，从第一份资料中搜索可能隐藏密码的位置。要考虑密码的所有排列可能性。

数据格式：

输入第一行：一个字符串s，全部由小写字母组成，长度小于1024*1024
紧接着一行是一个整数n,表示以下有n行密码，1<=n<=1000
紧接着是n行字符串，都是小写字母组成，长度都为8

要求输出：
一个整数, 表示每行密码的所有排列在s中匹配次数的总和。

例如：
用户输入：
aaaabbbbaabbcccc
2
aaaabbbb
abcabccc

则程序应该输出：
4

这是因为：第一个密码匹配了3次，第二个密码匹配了1次，一共4次。

资源约定：
峰值内存消耗（含虚拟机） < 512M
CPU消耗 < 5000ms

请严格按要求输出，不要画蛇添足地打印类似：“请您输入…” 的多余内容。

所有代码放在同一个源文件中，调试通过后，拷贝提交该源码。
注意：不要使用package语句。不要使用jdk1.7及以上版本的特性。
注意：主类的名字必须是：Main，否则按无效代码处理。
